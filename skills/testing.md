# Testing Skill

帮助编写和改进测试的技能。

## 使用场景

- 为新功能编写测试
- 提高测试覆盖率
- 修复失败的测试

## 提示词

```
请为 [功能名称] 编写测试，要求：

1. 测试覆盖
   - 正常情况
   - 边界情况
   - 异常情况

2. 测试类型
   - 单元测试
   - 集成测试（如需要）

3. 测试质量
   - 清晰的测试描述
   - 独立的测试用例
   - 使用 mock/stub（如需要）

使用项目现有的测试框架和风格。
```

## 常用命令

```bash
# 运行所有测试
npm test

# 运行特定文件测试
npm test -- user.service.spec.ts

# 查看覆盖率
npm run test:coverage

# 监听模式
npm run test:watch
```
