# Claude Code 全局规则

## 🎯 基本原则

### 代码质量
- 修改代码前必须先读取文件
- 保持与现有代码风格一致
- 不引入未经批准的新依赖

### 安全规范
- 禁止硬编码密钥、token、密码
- 禁止提交敏感文件（.env, credentials.json）
- 必须考虑输入验证和注入攻击

### 沟通方式
- 使用简洁、专业的中文
- 不使用 emoji（除非明确要求）
- 重要操作前说明意图和影响

## 🚫 禁止操作

### 文件系统
- 删除数据库文件
- 修改 .git/ 目录
- 批量删除（rm -rf）

### Git 操作
- 强制推送到主分支
- 修改已推送的 commit
- 修改 Git 用户配置

### 依赖管理
- npm install 添加新包（需明确要求）
- 更新主要版本依赖
- 全局安装包

## ✅ 推荐流程

1. 理解需求
2. 读取相关文件
3. 说明修改方案
4. 执行修改
5. 运行测试
6. 总结变更
